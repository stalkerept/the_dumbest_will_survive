**Императивное программирования** - парадигма прогаммирования (стиль написания исходного кода программы), для которой характерно следующее:
1. В исходном коде программы записываются инструкции (команды).
2. Инструкции должны выполняться последовательно.
3. Данные, получаемые при выполнении предыдущих инструкций, могут читаться из памяти последующими инструкциями.
4. Данные, полученные при выполнении иструкции, могут записываться в память.

То есть, **императивное программирование на С** - это описание того, как решить задачу с помощью последовательности команд, изменяющих состояние программы (переменные).

Императивная программа похожа на приказы, выражаемые повелительным наклонением в естественных языках, то есть представляют собой последовательность команд, которые должен выполнить процессор.

При императивном подходе к написанию кода (в отличии от функционального подхода ([[Функциональное программирование]])) широко используется присваивание. Наличие опреаторов присваивания увеличивает сложность модели вычислений и делает императивные программы подверженными специфическим ошибкам, не встречающимся при функциональном подходе.

Пример императивного программирования:
```c
#include <stdio.h>

int main() {
    int width = 5;      // Инструкция: задать состояние
    int height = 10;
    int area;

    area = width * height; // Инструкция: выполнить вычисление
    printf("Площадь: %d\n", area); // Инструкция: вывести результат
    return 0;
}
```
